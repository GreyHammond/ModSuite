01 -- The engine

Every message passes through eight filters.

First match stops the chain. All feed violations. Violations escalate to jail. Raids escalate to ban. The violation engine is the foundation -- everything plugs into it.

01

Invite filter

Blocks Discord invite links. Toggle with /automod invites.

02

Link filter

Domain whitelist/blacklist with subdomain matching and per-role/channel bypass.

03

Anti-phishing

Every URL checked against a live phishing database in real time. Fails open on timeout.

04

Word lists

Named deny lists. Whole-word for singles, substring for phrases. /wordlist group.

05

Message length

Min and max character thresholds. Empty messages (attachments only) skip min check.

06

All-caps filter

Configurable percentage threshold (default 70%). Ignores short messages.

07

Slowmode

Bot-enforced per-channel rate limit. Independent of Discord's built-in slowmode.

08

Spam detection

Message velocity, duplicate detection, mass mentions, emoji floods.

03 -- Getting started

Six steps to a running server.

Python 3.11+, a Discord bot token, and a few minutes. That's the shopping list.

  1. 01

    Create a Discord application

    Go to the Discord Developer Portal, create a new application, and add a bot. Copy the token.

    Under Bot > Privileged Gateway Intents, enable Server Members Intent and Message Content Intent.

  2. 02

    Clone & install

    git clone https://github.com/GreyHammond/ModSuite.git
cd ModSuite
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
  3. 03

    Configure environment

    cp .env.example .env
# Edit .env -- at minimum set DISCORD_TOKEN
# Optional: CORS_ORIGINS, API_HOST, API_PORT, Twitch keys, OAuth2
  4. 04

    Invite the bot with Administrator

    ModSuite creates roles, categories, and channels during /setup, so it needs Administrator. Generate the invite from Developer Portal > OAuth2 > URL Generator.

  5. 05

    Run the bot

    python bot.py

    The API starts automatically. Database auto-creates and auto-migrates on every startup.

  6. 06

    Run /setup in your server

    As a server admin, run /setup. Click Run Setup -- creates all default roles, channels, categories, and react-role messages. Then explore the 13 configuration panels to dial in your preferences.
