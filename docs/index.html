02 · Getting started

Six steps to a running server.

Python 3.11+, a Discord bot token, and a few minutes. That's the shopping list.

  1. 01

    Create a Discord application

    Go to the Discord Developer Portal, create a new application, and add a bot. Copy the token — you'll need it in step 4.

    Under Bot → Privileged Gateway Intents, enable Server Members Intent and Message Content Intent.

  2. 02

    Clone & install

    git clone https://github.com/GreyHammond/modsuite.git
cd modsuite/modsuite_v2
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
  3. 03

    Configure your token

    cp .env.example .env
# Edit .env and paste your bot token:
# DISCORD_TOKEN=your_token_here
  4. 04

    Invite the bot with Administrator

    ModSuite creates roles, categories, and channels during /setup, so it needs Administrator on the target server. Generate the invite from the Developer Portal → OAuth2 → URL Generator.

  5. 05

    Run the bot

    python bot.py

    This also starts the FastAPI backend on 127.0.0.1:8000 and serves the web dashboard.

  6. 06

    Run /setup in your server

    As a server admin, run /setup. Customise the messages if you like, then click Run Setup. ModSuite creates the color roles, Owner and Moderator roles, the ModMail category with #modmail / #mod-log / #closed-tickets, #self-roles with its react message, and the Jail category — then grants you the Owner role.

    Finish by running /automod status to review AutoMod defaults.

04 · Release history

Changelog.

v2.2.0

2026-07-06 Major

Added — AutoMod

  • Spam detection with message velocity, duplicate, mention, and emoji limits. Default action: delete + 10-minute timeout.
  • Link filtering with whitelist/blacklist modes and per-role/channel bypass. Preset lists included.
  • Discord invite filter as a separate toggle.
  • Immune roles for trusted members.
  • Filter chain: invites → links → spam. First match wins.
  • New /automod command group with 14 subcommands and a status dashboard.

Added — Raid Response upgrades

  • Active-raid auto-block for new joiners (kick or ban).
  • Auto-verification level bump during lockdown, restored on unlock.
  • Auto-unlock cooldown (default 5 min; 0 for manual-only).
  • Account age gate flags suspicious young accounts.
  • New /raidcfg command group.

Database

  • 22 new columns for AutoMod + raid config. Auto-migrated on next startup.

v2.1.0

2026-07-05 Minor

Added — Forum Thread Deletion

  • New /delete command lets forum-thread owners delete their own threads.
  • Two-step confirmation flow with permanent-warning modal.
  • Deletion logged to #mod-log with user, thread name, and timestamp.
  • Ownership enforced via thread.owner_id; buttons scoped to the invoker only.

Changed

  • Rebranded from "CommunityBot" to ModSuite in documentation and setup wizard.
  • README overhauled to document all 19 cogs, the API, and the web dashboard.

v2.0.0

2026-06 Major

Added — Web Stack

  • FastAPI REST API running in-process on 127.0.0.1:8000.
  • Vanilla-JS SPA dashboard — no build step, no framework.
  • Bot actions queue: web UI queues Discord actions, bot polls every 5s and executes.
  • Presence persistence across restarts.

Added — New Cogs

  • starboard, streamer, verify, remindme, reactmessage, move, reports.

v1.9.0

Baseline

Core cogs: setup, selfroles, modmail, moderation, warns, jail, userinfo, raid, panel, notes, admin, messages. Slash-command moderation, ModMail ticketing, basic raid detection, 14 color self-roles, SQLite persistence with auto-migrating schema.
